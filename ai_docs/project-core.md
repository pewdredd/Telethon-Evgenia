# Telethon-Evgenia: Ядро идеи проекта

## Суть продукта

Сервис автоматической квалификации и первичного контакта с лидами из Telegram-чатов.

### Цепочка

```
Парсер лидов (партнер) → Квалификация (AI) → Отправка сообщения (Telethon) → Переписка с лидом
```

1. **Партнер** парсит лиды из платных Telegram-чатов (уже существующий сервис)
2. **AI-квалификация** — фильтрует лидов, оставляя только релевантных
3. **Telethon-сервер** — отправляет персонализированное сообщение квалифицированному лиду от имени аккаунта пользователя
4. **Пользователь** продолжает переписку вручную

### Бизнес-модель

Продукт для продажи. Связка "парсер + квалификация + автоотправка" как готовое решение для лидогенерации через Telegram.

## Техническая архитектура

### Компоненты

```
n8n (оркестрация)
  │
  ├── Получает лиды от парсера партнера
  ├── Отправляет на AI-квалификацию
  └── Вызывает HTTP API для отправки сообщения
        │
        ▼
Telethon HTTP Server (этот проект)
  │
  ├── FastAPI — принимает HTTP-запросы
  ├── Telethon — отправляет сообщения через Telegram MTProto
  └── Session — авторизованный аккаунт пользователя
```

### HTTP API

**POST /send** — отправка сообщения лиду

```json
// Запрос
{
  "recipient": "@username",       // username или telegram user_id
  "message": "Персонализированный текст..."
}

// Ответ — успех
{
  "ok": true,
  "message_id": 12345
}

// Ответ — ошибка
{
  "ok": false,
  "error": "User privacy settings prevent sending messages"
}
```

**GET /health** — проверка что сервер жив и Telethon-сессия активна

```json
{
  "status": "ok",
  "authorized": true,
  "account": "@username"
}
```

### Стек

- **Python 3.11+**
- **FastAPI** — HTTP-сервер
- **Telethon** — Telegram MTProto клиент (юзербот)
- **uvicorn** — ASGI-сервер
- **n8n** — оркестрация (внешний, не часть этого проекта)

## Почему юзербот (Telethon), а не обычный бот

| | Обычный бот (Bot API) | Юзербот (Telethon) |
|---|---|---|
| Инициировать переписку | Нет — пользователь должен нажать /start | Да — можно написать первым |
| Выглядит как человек | Нет — метка "бот" | Да — обычный аккаунт |
| Доступ к чатам | Только если добавлен | Как обычный пользователь |
| Риск бана | Минимальный | Есть при массовой рассылке |

Для задачи "написать первым незнакомому человеку" — только юзербот.

## Идентификация получателя

Парсер лидов предоставляет данные о пользователе. Для отправки сообщения через Telethon можно использовать:

- **username** (`@username`) — самый надежный способ
- **user_id** (числовой Telegram ID) — работает всегда, если известен
- **phone** (номер телефона) — работает, только если номер есть в контактах

Парсер из чатов скорее всего дает **username** и/или **user_id**.

## Лимиты и защита от бана

### Известные ограничения Telegram

- ~20-50 новых контактов в день (точные лимиты скрыты и меняются)
- Спам-репорты от получателей ускоряют бан
- Одинаковый текст нескольким людям подряд = подозрение на спам
- Новые аккаунты банятся быстрее

### Меры защиты (реализовать в сервере)

1. **Rate limiting** — максимум N сообщений в день, настраивается
2. **Задержки** — случайная пауза 30-90 секунд между сообщениями
3. **Очередь** — сообщения ставятся в очередь, а не отправляются мгновенно
4. **Лимит в сутки** — жесткий лимит, после которого сервер отказывает в отправке до следующего дня
5. **Логирование** — все отправки записываются для анализа

### Рекомендуемые настройки по умолчанию

```
MAX_MESSAGES_PER_DAY = 25
MIN_DELAY_SECONDS = 30
MAX_DELAY_SECONDS = 90
```

## Флоу пользователя (клиента продукта)

### Первоначальная настройка

1. Клиент получает сервер (self-hosted или мы хостим)
2. Авторизует свой Telegram-аккаунт через Telethon (вводит телефон + код)
3. Настраивает n8n-воркфлоу: парсер → квалификация → отправка
4. Задает шаблон сообщения с переменными для персонализации

### Ежедневная работа

1. Парсер собирает лиды из целевых чатов
2. AI квалифицирует: подходит/не подходит, причина, рекомендуемое сообщение
3. Квалифицированные лиды получают персонализированное сообщение
4. Клиент видит ответы в своем Telegram и продолжает переписку вручную

## Что входит в этот проект (scope)

### MVP (первая версия)

- [ ] HTTP-сервер с эндпоинтом POST /send
- [ ] Авторизация Telethon-сессии (интерактивная, один раз)
- [ ] Отправка сообщения по username или user_id
- [ ] Rate limiting (лимит сообщений в день + задержки)
- [ ] GET /health — статус сервера и сессии
- [ ] Базовое логирование отправок
- [ ] Docker-контейнер для деплоя
- [ ] Конфигурация через .env файл

### Вне scope (на потом)

- AI-квалификация лидов (делается в n8n)
- Парсинг лидов (делает партнер)
- Веб-интерфейс
- Мульти-аккаунт (несколько Telegram-сессий)
- Автоответы на входящие
- Аналитика и дашборд
